                                Darf Nader • 4 months ago

                                `find` is still the most appreciated tool of all the core Unix V commands. No command has as much control flow power if interactive programs, it can be optimized based on search results and has a method for doing practically anything you can do to a file, and that isn't what you need, you can pipeline its output in a manner where it won't fail on special characters where normal posix control flow can be disrupted. If I could tell a Linux beginner that there was one problem to master first I would be find because it incorporates nearly all of the other crucial elements: grepping (patters and globs), stream editing (sed), subtleties of the file system, and even forking! Whenever I get an interview question on "hoe to program something" 4 out of 5 times I am able to use find to make a one line solutions. (With this, xargs is extremely powerful, but considering that -exec does this and with more control, you're better off "keeping it native".

                                I love you, find!
